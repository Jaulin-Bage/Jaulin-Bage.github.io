记录的园区

发表于 2023-09-15 更新于 2023-09-16 分类于 corgi

to begin with:

你说，整个博客有什么用呢？其实出发点很简单的，因为cip这个网站并不适合拿来做记录啊，而且似乎不是很吸引人。一个重要的原因就是他是wordpress做的，wordpress做出美观简洁的网站是可以的，但是比较困难。又因为是个人搭建的网站，受限于成本，租服务器只能选很便宜的，用起来会很卡顿。再加上经常收到奇怪的俄语评论，现在似乎大家不是很喜欢上去。于是我把记录放在这里，这样或许就能看到了。
从前的许多东西，比如放在双柯成基上，现在可以放这里。倒还不错。

或许我也可以拿来记录一些自己喜欢的东西吧。

发表于 2023-09-15 更新于 2023-09-16 分类于 record of anything
9.9.23

我已经完成了前三个步骤，包括登录github帐户、获取git的基本技术以及将远程存储库与本地主机关联。

测试文件可以从github上我自己的存储库中提取，尽管有些问题仍然存在，调试的痕迹将附在文档下面。

我使用的命令是“git pull origin master”，而正确的命令应该是“git pull origin main”，因为初始存储库中新分支的默认名称是“main”而不是“master”。

由于本地分支名称默认为“master”，因此作者编写了“master”。

url:
https://stackoverflow.com/questions/64628893/git-pull-master-with-error-fatal-couldnt-find-remote-ref-master

解析域名后，我可以通过简单的方式设置主题并生成信息，只需修改这些配置文件。
